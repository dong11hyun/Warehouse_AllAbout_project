#### 봇 탐지 차단 (Access Denied)
- User-Agent 변경 /  Referer 조작 / playwright-stealth 라이브러리 등을 시도했으나 실패.

> Chrome Remote Debugging (CDP) 방식을 도입. 
사용자가 직접 실행한 '진짜 크롬' 브라우저
--- 
#### 동적 렌더링 및 새 탭 이슈 
- 판매자 정보가 스크롤을 내려야만 로딩되는 Lazy Loading 방식임. 
- 상품 클릭 시 target="_blank"로 인해 새 탭이 열려 봇이 길을 잃음.
> mouse.wheel과 keyboard.press("End")를 조합
클릭(Click) 대신 URL을 수집 후 직접 이동하는 방식
--- 
#### 속도 최적화 (Sync → Async)
- 5건당 50초 (sync) => 5건당 5초 (async)
> Python asyncio / Async Playwright 도입
    Semaphore(5): 동시에 5개의 탭 병렬 수집
    Resource Blocking: 이미지, 폰트, 스타일시트 로딩을 차단
--- 
#### 데이터 무결성 / 동시쓰기
- 여러 개의 탭이 동시에 하나의 CSV 파일에 쓰기(Write)를 시도 후 데이터 유실 및 장애
> asyncio.Lock()을 사용하여 파일 저장 시점에는 한 번에 하나의 프로세스만 접근하도록 통제.
--- 
#### 차단 
- 웹 크롤링 시도 중 access denied 가 떳음
    - 링크를 시크릿모드에서 띄워보니 열림
    ip 차단은 아닌 것으로 생각
    쿠키 및 세션 초기화 진행 후 정상
> cdp 의 쿠키 유지
    쿠팡 매커니즘 예상
    1. 식별용 쿠키 발급
    2. 이상 징후 감지-쿠키 신뢰점수하락
    3. 세션 차단 / ip차단은 안함(여려명 쓰기때문에 차단신중?)
=> 쿠키 주기적 초기화
=> 봇 탐지 회피 로직 강화
