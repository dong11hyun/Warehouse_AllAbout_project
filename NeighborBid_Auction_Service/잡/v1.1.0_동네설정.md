## 🚀 Update v1.1: 지역 기반 필터링 및 계층형 조회 시스템

**개요:**
단순한 텍스트 검색을 넘어, **행정구역 계층 구조(시/도 > 시/군/구 > 읍/면/동)**를 기반으로 한 필터링 시스템을 구축했습니다. 상위 지역 선택 시 하위 지역의 매물을 자동으로 포함하여 사용자 경험(UX)을 개선했습니다.

### ✅ 구현 기능 상세
1.  **계층형 지역 모델링 (Hierarchical Region Model):**
    - `Self-Referencing ForeignKey`를 사용하여 무한 계층 구조 설계 (`parent` 필드 활용).
    - `Depth` 필드를 통해 행정구역 단계(시/구/동) 구분.

2.  **재귀적 필터링 로직 (Recursive Filtering):**
    - **문제:** 기존 필터는 '서울' 선택 시 정확히 '서울'로 등록된 물건만 조회하고, 산하의 '영등포구', '신길동' 물건을 누락하는 문제 발생.
    - **해결:** `get_all_descendants` 재귀 함수를 구현하여, 선택된 지역의 모든 하위(자식, 손자) 지역 ID를 수집 후 `IN` 쿼리로 조회.
    - **결과:** '서울특별시' 클릭 시 서울 산하의 모든 자치구 및 동네 매물을 한 번에 조회 가능.

3.  **자동 지역 태깅 시스템:**
    - 회원가입 시 사용자의 활동 지역(`Region`)을 설정.
    - 경매 물건 등록(`auction_create`) 시, 별도 선택 없이 사용자의 지역 정보를 자동으로 매핑하여 데이터 일관성 확보.

4.  **반응형 사이드바 UI:**
    - 사이드바에서 현재 선택된 지역을 하이라이팅.
    - `Depth`에 따른 들여쓰기(Indentation)를 적용하여 시각적 계층 구조 표현.
