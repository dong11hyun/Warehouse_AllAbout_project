## 기능 구현에 초점을 맞춘 초기 프로토타입

**A1 Auction**은 Django 프레임워크를 기반으로 구축된 **C2C 경매 플랫폼**
단순한 게시판 형태를 넘어, **실제 재화(Point/Money)의 흐름**과 **동시성 제어**가 포함된 입찰 시스템을 구현하였습니다. 현재 사용자가 직접 상품을 등록하고, 실시간으로 입찰하며, 낙찰 시 자동으로 정산(잠금/차감/입금)되는 핵심 거래 로직이 완성되어 있습니다.

#### 기술 스택

  - **Backend:** Python 3.11, Django 
  - **Database:** SQLite (개발용) 
  - **Frontend:** Django Templates, Bootstrap 5


#### 핵심 기능 (v1.0)

#### 1. 사용자 및 인증 (User & Auth)

  - [x] **커스텀 유저 모델:** `AbstractUser`를 확장하여 신용도(`reputation_score`) 및 닉네임 필드 추가
  - [x] **회원가입/로그인:** 일반 사용자용 가입 폼(`UserCreationForm` 커스텀) 및 로그인 시스템
  - [x] **접근 제어:** `@login_required` 데코레이터를 통한 비로그인 사용자 접근 제한

#### 2. 재화 및 지갑 시스템 (Economy)

  - [x] **전자 지갑(Wallet):** 회원가입 시 자동 생성, 잔액 관리
  - [x] **충전 시스템:** 마이페이지를 통한 재화 충전 시뮬레이션
  - [x] **이중 장부 로직:**
      - `balance` (사용 가능 금액)와 `locked_balance` (입찰 중 묶인 금액) 분리 운영
      - 입찰 시 즉시 차감되지 않고 `lock` 처리 → 낙찰 실패 시 `unlock` (환불)
  - [x] **트랜잭션 로그:** 모든 충전, 입찰, 환불, 정산 내역을 `Transaction` 모델에 기록

#### 3. 경매 및 입찰 엔진 (Auction Engine)

  - [x] **상품 등록:** 이미지 업로드 및 경매 시작/종료 시간 설정
  - [x] **동시성 제어:** `select_for_update`와 `transaction.atomic`을 활용한 비관적 락(Pessimistic Locking) 적용 (동일 금액 동시 입찰 방지)
  - [x] **입찰 로직:**
      - 상위 입찰 발생 시 기존 1등 입찰자에게 즉시 환불(Unlock) 및 알림
      - 최소 입찰 단위(`bid_unit`) 검증
  - [x] **즉시 구매(Instant Buy):** 설정된 즉시 구매가 지불 시 경매 조기 종료 및 즉시 정산

#### 4. 편의 및 커뮤니티 (UX/Community)

  - [x] **마이페이지:** 내 입찰 내역, 판매 내역, 지갑 잔액 관리 대시보드
  - [x] **검색 및 필터링:** 키워드 검색 및 정렬(최신순, 마감임박순, 가격순)
  - [x] **문의/댓글:** 상품 상세 페이지 내 Q\&A 기능
  - [x] **관심 상품(찜하기):** ManyToManyField를 활용한 위시리스트 구현
  - [x] **알림 시스템(Notification):** 입찰 경쟁, 낙찰 등 주요 이벤트 발생 시 웹페이지 상단 뱃지 알림
