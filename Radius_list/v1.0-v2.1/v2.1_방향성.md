# 프로젝트 방향성 검토: 위치 데이터 수집 및 분석

## 1. 기술 검토 및 한계점

### A. 카카오 API 이용
- **기능**: 특정 좌표(예: 다이소) 기준 반경 x 미터 내의 특정 카테고리(편의점, 카페 등)를 검색.
- **치명적 한계**: **최대 45개**(15개씩 3페이지)의 결과만 반환됨.
- **현실적인 문제**: 서울 강남역 등 밀집 지역은 반경 1km 내에 편의점/카페가 수백 개 존재하므로, API 사용 시 **"가장 가까운 45개"**만 수집되고 나머지는 누락됨.
- **판단**: 단순 주변 탐색 용도로는 적합하나, **완전한 상권 데이터 분석**에는 데이터 누락으로 인해 부적합함 (가능하지만 불완전).

### B. 구글 빅쿼리 (Google BigQuery)
- **빅쿼리**: 빅쿼리는 강력한 **데이터 분석/연산 도구(DB)**이지, 자체적으로 지도 데이터(상점 위치 등)를 보유한 서비스가 아님.
- **필수 전제 조건**: 분석을 위해서는 이미 **"전체 상점 좌표 데이터"**를 보유하고 있어야 함. (빅쿼리에 데이터를 넣고 쿼리하는 구조)
- **판단**: 편의점 위치 데이터가 없는 상태에서는 사용 불가능.

## 2. 해결 전략

상황에 따른 두 가지 현실적인 접근 방법입니다.

### 전략 1: 카카오 API 활용 (현재 방식 유지)
> **"데이터의 완벽함보다는 편의성을 추구할 때"**
- **접근**: 모든 데이터를 수집하는 것을 포기하고, **"반경 내 가장 가깝고 중요한 45개 매장"**만 지표로 삼음.
- **추천 대상**: 빠르고 간단하게 프로토타입을 만들거나 결과를 보고 싶은 경우 (초보자 추천).

### 전략 2: 공공데이터포털 활용 (정석 느낌)
> **"전체 데이터 분석이 필요할 때"**
- **접근**: [공공데이터포털 정보] 데이터를 다운로드하여 서울시 전체 카페/편의점 데이터를 확보.
- **구현**: 확보한 데이터를 내 DB(PostGIS 등)에 적재 후, DB 자체 공간 함수(ST_DWithin)를 통해 쿼리.
- **장점**: 데이터 누락 없이 정확한 전체 상권 분석 가능.

## 3. 빅쿼리를 활용한 대용량 분석 시나리오

데이터 제한 없이 빅쿼리의 성능을 활용?

1.  **데이터 확보**: 공공데이터포털에서 "서울시 휴게음식점 인허가 정보" CSV 다운로드
2.  **전체 데이터 적재**: 확보한 CSV를 빅쿼리 테이블로 업로드
3.  **타겟 데이터 적재**: 수집한 다이소 좌표 데이터를 CSV로 만들어 빅쿼리에 업로드
4.  **분석 수행**: 빅쿼리 SQL 콘솔에서 `ST_DISTANCE` 등의 지리 공간 함수를 사용하여 두 테이블 조인 및 분석
    > 해당 시나리오는 구상단계 및 방향성 정하기에서 세운 시나리오
