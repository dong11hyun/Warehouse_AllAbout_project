## 🏪 다이소 상권 분석 프로젝트 (Daiso Research)
- 특정 상권(다이소)을 중심으로 반경 1km~5km 내의 소상공인 데이터를 카카오 로컬 API를 통해 수집하고 DB화하는 프로젝트
#### 배경
- 중소기업벤처부 리서치 업무의 일환으로, 특정 상권(다이소) 주변의 소상공인 리스트를 조사해야 하는 업무.
- 수작업으로 지도 데이터를 검색하고 정리하는 것은 비효율적이며 데이터의 정합성을 보장하기 어려움

--- 

- **API 활용:** Kakao REST API를 연동하여 정확한 좌표 기반의 반경 검색(Radius Search)을 구현했습니다.
- **데이터 무결성 확보:** 중복 수집 방지 로직을 적용하여 동일 상권 데이터의 중복 적재를 방지했습니다.

---


#### Tech Stack
- **Framework:** Django 5.2 (Python 3.10+)
- **Database:** SQLite (개발/테스트 용이성), 확장 시 PostgreSQL 전환 고려
- **External API:** Kakao Maps REST API
- **Environment:** `python-dotenv`를 통한 환경 변수 관리

#### Core Logic (`collect_cafes.py`)

API를 활용해 주소를 좌표로 변환하고, 지정 카테고리 주변 데이터를 페이지네이션·레이트리밋을 고려해 수집하며 중복을 방지해 저장

---

#### 코드 리뷰

- 수집 로직을 View가 아닌 Management Command로 분리하여, 웹 요청 타임아웃과 무관하게 배치(Batch) 작업으로 실행 가능.
- 확장성 있는 모델 설계: 추후 다른 브랜드 분석으로 확장 용이

##### 개선 및 확장성 (Scalability & Improvements)
- `DAISO_TARGETS` 리스트 하드코딩.
> `TargetStore` 모델을 별도로 생성하여 관리자 페이지(Admin)에서 분석 대상을 동적으로 추가/관리하도록 정규화(Normalization) 필요.

- `requests`를 사용한 동기(Synchronous) 방식. 타겟이 늘어날수록 실행 시간이 선형적으로 증가.

>  `Celery` 또는 `Asyncio`를 도입하여 다수의 타겟을 병렬로 수집하여 성능 개선 가능

- 수집 반경(1000m), 타겟 카테고리
> `settings.py` 또는 DB 설정으로 분리하여 코드 수정 없이 파라미터를 변경할 수 있도록 개선..?
