#### 외부 API의 제약 사항 

- Kakao Local API는 한 번의 요청에 모든 데이터를 제공하지 않았음.
- 짧은 시간에 과도한 요청 => 에러 발생

>루프 구현 1페이지(15개) * 3번
`time.sleep(0.2)` => API 서버에 과부하 줄여봄.

---

#### 데이터 무결성 
- 스크립트 재실행 DB 중복 적재
 > 로직을 애플리케이션 레벨 구현
`NearbyStore.objects.filter`
<확장>
> -DB 모델의 `Meta` 클래스에 `unique_together = ('name', 'address')` 제약 조건을 추가
-Django의 `update_or_create()` 메서드 활용
-Upsert(Update + Insert) 로직으로 리팩토링
