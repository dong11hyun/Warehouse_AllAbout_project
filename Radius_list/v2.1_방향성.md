## A. 카카오 API 이용 (판단:  가능하지만 불완전함)
기능: 특정 좌표(다이소) 기준 반경 x 미터 내의 카테고리(편의점 CS2, 카페 CE7) 검색이 가능합니다.

치명적 단점 (45개 제한): 카카오 API는 한 번 검색에 **최대 45개(15개씩 3페이지)**까지만 결과를 보여줍니다.

현실: 서울 강남역 다이소 반경 1km 안에는 편의점과 카페가 수백 개가 넘습니다. 카카오 API를 쓰면 **"가장 가까운 45개"**만 가져오고 나머지는 잘립니다.

결론: "주변에 뭐가 있나?" 정도는 괜찮지만, "모두 찾아서 상권 분석"을 하기엔 데이터가 빕니다.

---

## B. 구글 빅쿼리 (판단: X 데이터가 없음)

오해: 빅쿼리는 엄청나게 빠른 **계산기(DB)** 이지, 지도 데이터(편의점 위치 정보)를 가지고 있는 지도 서비스가 아닙니다.

필요조건: 빅쿼리에서 분석하려면, 님께서 이미 **"서울시 전체 편의점/카페 좌표 데이터(수만 개)"**를 가지고 있어야 합니다. 그 데이터를 빅쿼리에 넣고 다이소 좌표랑 매칭하는 것입니다.

결론: 편의점 위치 데이터를 수집하기 전에는 빅쿼리는 무용지물입니다.

---

## 해결 전략
가장 현실적인 방법은 두 가지입니다.

카카오 API 활용 (현재 방식 유지): "모두"는 포기하고, **"반경 내 가장 가까운/중요한 45개 매장"** 만 수집해서 지표로 삼는다. (초보자에게 추천)

공공데이터포털 활용 (고급): [소상공인시장진흥공단_상가(상권)정보] 데이터를 다운받아 서울시 전체 카페/편의점 데이터를 내 DB(PostGIS)에 다 넣고, DB 자체 기능(ST_DWithin)으로 쿼리한다. (이게 정석입니다.)
---

#### 빅쿼리를 꼭 쓰고 싶다면?

만약 "나는 45개 제한이 너무 싫고, 빅쿼리로 대용량 분석을 하고 싶다"면 흐름은 이렇습니다.

공공데이터포털에서 "소상공인시장진흥공단_상가(상권)정보" CSV 다운로드 (몇백 MB).

이 CSV를 빅쿼리 테이블로 업로드 (seoul_all_stores).

현재 수집한 다이소 데이터를 CSV로 뽑아서 빅쿼리에 업로드 (my_daiso).

빅쿼리 SQL 콘솔에서 ST_DISTANCE 함수를 써서 조인.

이 방법은 데이터 수집(ETL) 파이프라인이 복잡
